Architecture · Intégration Odoo

Zéro CSS custom — la maquette est XML natif

08
1

Tokens marque

Palette bleu assurance + statuts paid/unpaid/pending + échelle d'encre. Définis en variables SCSS.
  • $ac-blue-600 · $ac-success · $ac-danger
  • $ac-ink-100 → $ac-ink-900
  • $radius-sm · $radius · $radius-lg
2

Bootstrap 5

Les tokens cascadent vers Bootstrap via les surcharges $primary, $border-radius, $body-bg.
  • $primary → $ac-blue-600
  • $body-bg → $ac-paper
  • $box-shadow · $border-radius-lg
3

Hooks Odoo

Odoo lit ses propres variables SCSS — il suffit d'aligner les noms pour habiller tout le backend.
  • $o-brand-primary · $o-action
  • $o-form-sheet-radius · $o-form-sheet-bg
  • $o-statusbar-active-bg
  • $o-kanban-record-radius
  • $o-button-box-radius · $o-chatter-background
4

Vues XML

Le développeur Odoo n'écrit que du XML standard — toutes les vues présentées sont reproductibles tel quel.
  • <form> <header> <sheet>
  • <div class="oe_button_box">
  • <notebook> <page>
  • <chatter />
  • <tree default_group_by="…">
__manifest__.py — un seul prepend suffit
'assets': {
    'web._assets_primary_variables': [
        ('prepend', 'assurcourtage/static/src/scss/_variables.scss'),
    ],
}
